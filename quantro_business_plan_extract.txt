03
Problem & Market Opportunity
Most retail traders do not fail because they lack strategy. They fail because they cannot execute their strategy
consistently. Emotional interference — fear, greed, hesitation, overtrading — is the primary driver of retail trading
losses. Research consistently shows that over 70% of retail traders cite emotional decision-making as a key failure
factor. The problem is not information. The problem is execution.
The Five Core Problems
 PROBLEM
 CONSEQUENCE
Emotional Execution
Traders abandon their systems at the worst possible moment — under pressure,
when consistency matters most.
Time Dependency
Profitable systems require constant screen monitoring. Most traders have full
lives that preclude this.
Inconsistency
Same trader, same system, different results — because human execution
introduces variance.
Custodial Risk
Existing bot platforms often hold user funds or require fund transfers, creating
trust and security risk.
Complexity Barrier
Technical bot setup exceeds the skill level of most skilled traders, blocking
adoption of automation.

Technology & Infrastructure
Quantro's technical architecture is built for security, scalability, and reliability by Spencer, co-founder with deep
experience in financial system architecture. The platform is non-custodial by design — not as an afterthought.
Security is not a feature. It is the foundation.
Architecture Principles
 Non-custodial by design: Quantro never holds, moves, or controls user funds under any circumstance
 API key security: all connections use withdrawal-disabled, read/trade-only keys
 AES-256 encryption at rest and in transit for all user credentials
 Execution isolation: each user's bot runs in an isolated execution environment
 Immutable audit trails: all execution events logged, timestamped, and tamper-proof
 Redundancy: no single point of failure in the execution pipeline

Key Definitions
 TERM
 DEFINITION
Non-Custodial
Platform never holds, moves, or controls user funds. Users connect own accounts via
API.
Performance Fee
A percentage of profits generated by the bot, charged monthly. Only applies when the
system profits.
ARPU
Average Revenue Per User per month — includes subscription, performance fee, and
trade fees.
CAC
Customer Acquisition Cost — total marketing cost divided by new customers acquired.
LTV

period.
ARR

income.
